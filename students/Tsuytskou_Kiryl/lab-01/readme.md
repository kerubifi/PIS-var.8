# Описание Use-case: Размещение объявления

**Первичный актор**: Продавец (Авторизованный пользователь)

**Цель**: Разместить объявление о продаже товара, сделав его доступным для покупателей, обновив связанные категории и уведомив модераторов.

**Предусловия**: 
- Продавец авторизован в системе.
- Существует объект "Черновик объявления" с заполненным названием (минимум 5 символов) и описанием (максимум 5000 символов).

**Основной поток (Happy Path)**:
1. Продавец открывает страницу создания объявления.
2. Продавец нажимает кнопку «Разместить на модерации».
3. Система проверяет заполненность обязательных полей (название, описание, цена >= 0).
4. Система создает объявление со статусом `PENDING_MODERATION` и фиксирует дату создания.
5. Система сохраняет связи объявления с выбранными категориями.
6. Система инвалидирует (сбрасывает) кэш списка объявлений, чтобы новое объявление появилось в ленте.
7. Система инициирует отправку уведомления модератору о новом объявлении на проверку.
8. Система отображает сообщение об успешном размещении на модерации и перенаправляет на страницу объявления.

**Постусловия**:
- Объявление доступно по прямой ссылке и в общем списке (после модерации).
- Модератор получил уведомление о необходимости проверки (в течение некоторого времени).
- Счетчик объявлений продавца увеличился на 1.

**Альтернативные потоки**:
- **3a. Недостаточное количество символов в названии**:
  - 3a1. Система выводит предупреждение «Название должно содержать минимум 5 символов».
  - 3a2. Кнопка размещения остается заблокированной до исправления.
- **3b. Превышение лимита описания**:
  - 3b1. Система выводит предупреждение «Описание не должно превышать 5000 символов».
  - 3b2. Кнопка размещения остается заблокированной до исправления.
- **3c. Отрицательная цена**:
  - 3c1. Система выводит предупреждение «Цена не может быть отрицательной».
  - 3c2. Кнопка размещения остается заблокированной до исправления.
- **3d. Бесплатное объявление**:
  - 3d1. Система принимает цену 0 как допустимую.
  - 3d2. Объявление создается с флагом is_free = True.

**Исключительные ситуации**:
- **4a. Сбой базы данных при сохранении**:
  - 4a1. Система откатывает изменения статуса.
  - 4a2. Выводится ошибка «Технические шоколадки, попробуйте позже».
- **7a. Таймаут сервиса уведомлений**:
  - 7a1. Объявление создается успешно.
  - 7a2. Задача на уведомление ставится в очередь на повтор (Retry).
